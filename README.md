## Установка

Python >= 3.10

### Установка виртуального окружения
```shell
python -m venv .venv
```

## Настройка переменных среды
Создать файл `.env` и скопировать в него содержимое `.env.example`.

В файле `.env` заполнить переменные среды:
- UI_USERNAME - имя пользователя для входа в учётную запись
- UI_PASSWORD - пароль пользователя для входа в учётную запись
- API_TOKEN - токен авторизации

### Активация виртуального окружения

Windows
```shell
./.venv/Scripts/activate
```

Linux или MacOS
```shell
source ./.venv/bin/activate
```

### Установка зависимостей
```shell
pip install -r requirements.txt"
```

## Запуск автотестов
```shell
pytest test_ui -s -v --headed (для запуска в интерфейсе)
pytest test_service_api.py -s -v
```

## Структура тестов
### UI-тесты
Авторизация:
успешная и неуспешная (неверный логин/пароль, пустые поля)

Форма услуг:
проверка наличия всех элементов
позитивные граничные значения для price, quantity, name
проверка расчёта НДС и итоговой суммы
негативные сценарии (ошибки валидации, пустые/некорректные поля)

CRUD операции:
создание, редактирование, удаление услуг

### API-тесты

Создание услуги:
- позитивные сценарии с корректными данными  
- проверка расчёта НДС и итоговой суммы  
- реалистичные данные и граничные значения  

Получение услуги:
- существующие
- несуществующие
- некорректные ID

Обновление услуги:
- корректное обновление
- изменение цены
- проверка пересчёта НДС
- попытка обновления несуществующей услуги

Удаление услуги:
- успешное удаление
- двойное удаление
- удаление несуществующей услуги

Негативные сценарии:
- отсутствие авторизации / неверный токен
- некорректные типы данных
- пустые поля
- превышение лимитов
- отрицательные
- нулевые и переполняющие значения
